# ESP32-S3 NFC 音乐盒

基于 ESP32-S3-DevKitC-1 的 NFC 刷卡点歌音乐盒。使用 GC9A01 圆形屏显示黑胶唱片旋转动画，PN532 识别 NFC 标签，ZK-502C 功放驱动四只扬声器，音乐存储在 SD 卡中，由 12V 锂电池供电，并支持蓝牙模式切换。

## 功能列表

| 功能 | 说明 |
|------|------|
| NFC 刷卡选歌 | 每张 NFC 卡对应一首歌，刷卡立即播放 |
| SD 卡顺序播放 | 按键控制上一首 / 下一首 / 播放 / 暂停 |
| 断点续播 | 暂停后恢复从中断位置继续，不从头播放 |
| 黑胶唱片动画 | 旋转专辑封面（BMP）+ 唱片纹路光效 + 唱针臂 |
| 蓝牙模式 | 长按播放键切换；ZK-502C 板载 BT 接收手机音频 |
| 音量浮层 | 转动编码器调节音量时屏幕弹出大字音量 |
| 电量显示（外置）| 独立 LED 电量指示模块，直接接电池，无需 GPIO |

## 🤔 常见问题：必须要 DAC 模块吗？

**短答案：不一定需要买模块，但必须有某种方法把数字信号转为模拟信号。**

### 为什么需要转换？

| 设备 | 音频接口类型 |
|------|------------|
| ESP32-S3 | **只有数字输出**（I2S）；与原版 ESP32 不同，S3 没有内置 DAC |
| ZK-502C（TPA3116D2）| **只接受模拟输入**（AUX 3.5mm 口或板载蓝牙） |

两者之间存在"数字↔模拟"的鸿沟，必须跨越它。

> **关于 ZK-502C 的蓝牙：** 该模块使用经典蓝牙（A2DP）。ESP32-S3 只有 BLE，不兼容 A2DP，因此 ESP32 无法直接向 ZK-502C 发流。蓝牙模式下，用户的手机直接连接 ZK-502C 板载蓝牙模块，ESP32 负责界面显示和切换。

---

### 两种方案对比

| | 方案 A（推荐）| 方案 B（无模块）|
|-|-------------|----------------|
| **原理** | I2S 数字信号 → PCM5102A DAC → 模拟输出 | I2S PDM-TX 数字信号 → RC 低通滤波 → 模拟输出 |
| **额外器件** | PCM5102A 模块（约 ¥15–25）| 1kΩ 电阻 ×2 + 10nF 电容 ×2（约 ¥0.5）|
| **音质** | 优（SNR >100dB，立体声）| 良（底噪略高，单声道）|
| **接线复杂度** | 简单（3 根信号线）| 简单（1 根信号线 + RC 滤波器）|
| **固件修改** | 无需（默认）| 修改 `config.h` 一行 |
| **适合场景** | 追求音质、已有模块 | 手边无模块、想快速验证 |

---

### 方案 A：PCM5102A I2S DAC（推荐）

这是默认方案，固件无需修改。

**接线：**

| PCM5102A 引脚 | ESP32-S3 GPIO |
|-------------|--------------|
| BCK (BCLK)  | GPIO 15      |
| LCK (LRCK)  | GPIO 16      |
| DIN (DATA)  | GPIO 17      |
| VCC         | 3.3V         |
| GND         | GND          |
| FLT         | GND          |
| DEMP        | GND          |
| XSMT        | 3.3V（静音控制，高电平 = 工作）|
| FMT         | GND          |

PCM5102A 的 LOUT/ROUT → ZK-502C 的 L-IN/R-IN。

---

### 方案 B：PDM 输出 + RC 低通滤波（无需 DAC 模块）

只需 **2 个电阻 + 2 个电容**（共约 ¥0.5），ESP32-S3 GPIO 14 直接驱动滤波器后送入 ZK-502C AUX。

**修改 `config.h`（只改一行）：**

```cpp
#define AUDIO_MODE  AUDIO_MODE_PDM   // 将默认的 AUDIO_MODE_I2S_DAC 改为此行
```

**硬件电路（2 级 RC 低通滤波器）：**

```
ESP32-S3
GPIO 14 ──┤ R1 1kΩ ├──┬──┤ R2 1kΩ ├──┬── ZK-502C L-IN
                       │              │   （同时接 R-IN，双声道播相同内容）
                      C1             C2
                     10nF           10nF
                       │              │
                      GND            GND
```

截止频率 ≈ 1/(2π × 1kΩ × 10nF) ≈ **16kHz**（足以覆盖人耳听觉范围）。

> **注意**：PDM 方案输出为单声道，L-IN 和 R-IN 接同一信号，
> 喇叭仍然各自独立工作（左右声道内容相同）。

## 🤔 常见问题：需要 5V→3.3V 降压吗？

**短答案：不用额外买。** 本方案的主电源是 12V → 5V DC-DC（见硬件清单），而 **ESP32-S3-DevKitC-1 板载 3.3V LDO**，可以直接给 GC9A01、PN532、SD 模块等 3.3V 设备供电。只要：

- 主供电输入 DevKit 的 5V（来自 12V→5V 降压模块或 USB 5V）
- 所有 3.3V 负载接 DevKit 的 3V3 引脚

就不需要再加一颗独立的 5V→3.3V 降压器。**请勿把 5V 直接接到屏幕/模块的 3.3V 引脚。**

## 引脚分配

### GC9A01 圆形屏（SPI）

| 屏幕引脚 | ESP32-S3 GPIO |
|---------|--------------|
| MOSI    | GPIO 11      |
| SCLK    | GPIO 12      |
| CS      | GPIO 10      |
| DC      | GPIO 9       |
| RST     | GPIO 8       |
| BL      | GPIO 46      |
| VCC     | 3.3V         |
| GND     | GND          |

> **如果买到 6Pin/省略背光的 GC9A01 模块**：常见只引出 `VCC / GND / SCL / SDA / DC / RST (+CS 焊盘)`，背光已焊死到 VCC。请按下表接线，确保仍然有 **独立 CS**，否则与同总线的 SD 卡会互相干扰。
>
>| 屏幕丝印 | 连接到 ESP32-S3 |
>|----------|----------------|
>| VCC      | 3.3V（不要接 5V）|
>| GND      | GND |
>| SCL / CLK| GPIO 12（SCLK）|
>| SDA / DIN| GPIO 11（MOSI）|
>| DC / D/C | GPIO 9 |
>| RES / RST| GPIO 8 |
>| CS / SS  | GPIO 10（若仅有背面焊盘，请飞线引出；若被固定为低电平，会与 SD 卡冲突）|
>| BL / LED | 若无此引脚可留空，ESP32 侧 GPIO 46 也可以悬空；如有则接 GPIO 46（或直接 3.3V 常亮）|
>
> 提示：如模块仅在背面留有 CS 焊盘，需飞线引出并保持可拉高，避免与同总线的 SD 卡互相占用。

> **如果买到 12Pin GC9A01 模块**： 常见丝印顺序为 `GND / LEDK / LEDA / VDD / GND / GND / D/C / CS / SCL / SDA / RESET / GND` （对应图示中的 1–12 号脚）。 接线示例：
>
>| NO. | 屏幕丝印 | 连接到 ESP32-S3 | 说明 |
>|-----|----------|----------------|------|
>| 1/5/6/12 | GND | GND            | 多个地脚任选其一，推荐都接地 |
>| 2   | LEDK     | GND（默认） | 背光负极，默认接地 = 常亮。如需可控亮，详见下方“补充”段落，用三极管/小 MOS 反相，由 GPIO46 高电平拉低导通。 |
>| 3   | LEDA     | 3.3V           | 背光正极；若模块无板载限流电阻，建议串 100–220Ω 限流，否则遵从板载电阻规格 |
>| 4   | VDD      | 3.3V           | **仅接 3.3V，勿接 5V**（与 6Pin 模块同理） |
>| 7   | D/C      | GPIO 9         | 数据/命令 |
>| 8   | CS       | GPIO 10        | SPI 片选信号，**低电平有效**；需独立 CS 以免与 SD 总线冲突 |
>| 9   | SCL      | GPIO 12（SCLK）| SPI 时钟 |
>| 10  | SDA      | GPIO 11（MOSI）| SPI 数据 |
>| 11  | RESET    | GPIO 8         | 复位信号，**低电平有效**（与 CS 逻辑一致） |

> 补充：固件默认把 GPIO46（BL）拉高点亮背光，适用于带 BL/LEDA 的正逻辑模块。若 12Pin 模块只露出 LEDK（负极），最简方案是把 LEDK 接地常亮；如需可控，可用 NPN/小 MOS 管做反相，由 GPIO46 高电平驱动导通以拉低 LEDK（对应上表 NO.2 的 LEDK “可控亮”方案）。

### 音频输出引脚（二选一）

**方案 A：PCM5102A DAC（`AUDIO_MODE_I2S_DAC`，默认）**

| PCM5102A 引脚 | ESP32-S3 GPIO |
|-------------|--------------|
| BCK (BCLK)  | GPIO 15      |
| LCK (LRCK)  | GPIO 16      |
| DIN (DATA)  | GPIO 17      |
| FLT / DEMP / FMT | GND   |
| XSMT        | 3.3V         |
| VCC         | 3.3V         |

**方案 B：PDM + RC 滤波（`AUDIO_MODE_PDM`，无需模块）**

| 信号 | ESP32-S3 GPIO |
|------|--------------|

### ZK-502C 功放供电与连接

| ZK-502C 引脚 | 连接目标 |
|------------|---------|
| VCC（12V）  | 12V 电池正极 |
| GND         | 公共地  |
| L-IN / R-IN | PCM5102A LOUT/ROUT（方案 A）或 PDM 滤波输出（方案 B）|
| L+ L−       | 全频喇叭（4Ω 10W）|
| R+ R−       | 全频喇叭（4Ω 10W）|

> 高音喇叭（4Ω 11W）通过 **4.7μF 无极性电容** 串联后并联在全频喇叭输出端，起高通分频作用。

### 电源连接

```
12V 电池 ──┬──→ ZK-502C VCC（直接 12V 供电）
           ├──→ 12V→5V 降压模块 VIN
           │         │
           │         └──→ 5V 输出 ──→ ESP32-S3 5V 引脚
           │                         ├──→ PCM5102A（方案 A，3.3V 由 LDO 提供）
           │                         └──→ PN532 / SD 卡模块
           │
           └──→ 外置电量指示模块（B+ 接电池正，B− 接电池负，仅 2 根线）
```

## 操作说明

| 操作 | 动作 |
|------|------|
| **NFC 刷卡** | 将 NFC 标签靠近 PN532 → 立即播放对应歌曲 |
| **播放/暂停** | 短按 PIN_BTN_PLAY（GPIO48）|
| **上一首** | 短按 PIN_BTN_PREV（GPIO21）|
| **下一首** | 短按 PIN_BTN_NEXT（GPIO47）|
| **切换蓝牙模式** | **长按** PIN_BTN_PLAY（≥1.5 秒）|
| **音量调节** | 旋转编码器（GPIO18/19）；屏幕弹出音量大字 |
| **退出蓝牙模式** | 再次长按 PIN_BTN_PLAY |

### 蓝牙使用流程
1. 长按播放键 → 屏幕显示"蓝牙模式"
2. 将 ZK-502C 功放拨到 **BT 挡**（板上物理开关）
3. 手机蓝牙搜索 → 连接"ZK-502C"（或 ZK502C 等设备名）
4. 手机正常播放音乐，通过 ZK-502C 板载蓝牙输出
5. 再次长按播放键 → 退出蓝牙模式，恢复 SD 卡播放

### 专辑封面
- 在 SD 卡 `/cover/` 目录放置 **100×100 像素 24-bit BMP** 文件
- 文件名与歌曲 ID 对应：`001.bmp` 对应 `001.mp3`
- 无封面时自动显示旋转色块占位图案

## 注意事项

1. **共地**：12V 电池、降压模块、ESP32-S3、ZK-502C 必须共地。
2. **电流**：降压模块选用至少 3A 输出，ZK-502C 空载约 300mA，满功率可达 3A+。
3. **散热**：ZK-502C（TPA3116D2）在大音量下会发热，建议加散热片。
4. **NFC 天线距离**：PN532 读卡距离约 3-5cm，保持标签与模块平行。
5. **SD 卡格式**：使用 FAT32 格式，文件名限 8.3 格式（`001.mp3`）。
6. **音量控制**：ZK-502C 板上有电位器可调整最大音量；固件通过 I2S 音量调节软件音量。
